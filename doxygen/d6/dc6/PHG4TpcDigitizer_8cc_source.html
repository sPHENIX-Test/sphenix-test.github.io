161  // Digitizes the Tpc cells that were created in PHG4CylinderCellTpcReco
162  // These contain as edep the number of electrons out of the GEM stack, distributed between Z bins by shaper response and ADC clock window
163  // - i.e. all of the phi and Z bins in a cluster have edep values that add up to the primary charge in the layer times 2000
164 
165  // NOTES:
166  // Modified by ADF June 2018 to do the following:
167  // Add noise to cells before digitizing
168  // Digitize the first adc time bin to exceed the threshold, and the 4 bins after that
169  // If the adc value is still above the threshold after 5 bins, repeat for the next 5 bins
170 
171  // Electron production:
172  // A MIP produces 32 electrons in 1.25 cm of Ne:CF4 gas
173  // The nominal GEM gain is 2000 => 64,000 electrons per MIP through 1.25 cm gas
174  // Thus a MIP produces a charge value out of the GEM stack of 64000/6.242x10^18 = 10.2 fC
175 
176  // SAMPA:
177  // See https://indico.cern.ch/event/489996/timetable/#all.detailed "SAMPA Chip: the New ASIC for the ALICE Tpc and MCH Upgrades", M Bregant
178  // The SAMPA has a maximum output voltage of 2200 mV (but the pedestal is about 200 mV)
179  // The SAMPA shaper is set to 80 ns peaking time
180  // The ADC Digitizes the SAMPA shaper output into 1024 channels
181  // Conversion gains of 20 mV/fC or 30 mV/fC are possible - 1 fC charge input produces a peak volatge out of the shaper of 20 or 30 mV
182  // At 30 mV/fC, the input signal saturates at 2.2 V / 30 mV/fC = 73 fC (say 67 with pedestal not at zero)
183  // At 20 mV/fC, the input signal saturates at 2.2 V / 20 mV/fC = 110 fC (say 100 fC with pedestal not at zero) - assume 20 mV/fC
184  // The equivalent noise charge RMS at 20 mV/fC was measured (w/o detector capacitance) at 490 electrons
185  // - note: this appears to be just the pedestal RMS voltage spread divided by the conversion gain, so it is a bit of a funny number (see below)
186  // - it is better to think of noise and signal in terms of voltage at the input of the ADC
187  // Bregant's slides say 670 electrons ENC for the full chip with 18 pf detector, as in ALICE - should use that number
188 
189  // Signal:
190  // To normalize the signal in each cell, take the entire charge on the pad and multiply by 20 mV/fC to get the adc input AT THE PEAK of the shaper
191  // The cell contents should thus be multipied by the normalization given by:
192  // V_peak = Q_pad (electrons) * 1.6e-04 fC/electron * 20 mV/fC
193  // From the sims, for 80 ns and 18.8 MHz, if we take the input charge and spread it a across the shaping time (which is how it has always been done, and is
194  // not the best way to think about it, because the ADC does not see charge it sees voltage out of a charge integrating preamp followed by a shaper), to get
195  // the voltage at the ADC input right, then the values of Q_(pad,z) have to be scaled up by 2.4
196  // V_(pad,z) = 2.4 * Q_(pad,z) (electrons) * 1.6e-04 fC/electron * 20 mV/fC = Q_(pad,z) * 7.68e-03 (in mV)
197  // ADU_(pad,z) = V_(pad,z) * (1024 ADU / 2200 mV) = V_(pad,z) * 0.465
198  // Remember that Q_(pad,z) is the GEM output charge
199 
200  // Noise:
201  // The ENC is defined as the RMS spread of the ADC pedestal distribution coming out from the SAMPA divided by the corresponding conversion gain.
202  // The full range of the ADC input is 2.2V (which will become 1024 adc counts, i.e. 1024 ADU's).
203  // If you see the RMS of the pedestal in adc counts as 1 at the gain of 20mV/fC, the ENC would be defined by:
204  // (2200 [mV]) * (1/1024) / (20[mV/fC]) / (1.6*10^-4 [fC]) = 671 [electrons]
205  // The RMS noise voltage would be: V_RMS = ENC (electrons) *1.6e-04 fC/electron * 20 mV/fC = ENC (electrons) * 3.2e-03 (in mV)
206  // The ADC readout would be: ADU = V_RMS * (1024 ADU / 2200 mV) = V_RMS * 0.465
207 
208  // The cells that we need to digitize here contain as the energy "edep", which is the number of electrons out of the GEM stack
209  // distributed over the pads and ADC time bins according to the output time distribution of the SAMPA shaper - not what really happens, see above
210  // We convert to volts at the input to the ADC and add noise generated with the RMS value of the noise voltage at the ADC input
