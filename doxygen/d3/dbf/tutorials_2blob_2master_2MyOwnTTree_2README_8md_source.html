Go to the documentation of this file.
1 This directory contains 2 examples to create your own TTree using a class (and
2 Fun4All). The sources are in ./src, the macros to run the show are in ./macros.
3 
4 The MySimpleTree class is stored in a TTree by itself and then inside a
5 TClonesArray (just to show you that the MySimpleTree class doesn't need any
6 modifications for the TClonesArray).
7 
8 Unlike our DST's there is no virtual base class to keep old files readable
9 after you change the content of your class. It's not difficult to add this
10 feature but I wanted to keep it straight and simple.
11 
12 Library build:
13 Don't build the library in the source area - it can be done but you do not do
14 yourself any favors with this. Create a build area and think of an install area location (the install area will be created during "make install", you need to have write access). Go to the build area and
15 
16 <srcdir>/autogen.sh --prefix=<install area>
17 make install
18 
19 To run the macros you have to add your install area to the LD_LIBRARY_PATH and
20 to the ROOT_INCLUDE_PATH (where root6 looks for includes mentioned in the
21 macro). For the ROOT_INCLUDE_PATH we have a setup script which will add your install area but also the includes in OFFLINE_MAIN (and G4 and ROOT).
22 
23 setenv LD_LIBRARY_PATH <install area>/lib:$LD_LIBRARY_PATH
24 source /opt/sphenix/core/bin/setup_root6.csh <install area>
25 
26 With these settings you can run the macros in the macro directory.
