367 // all set_default_XXX_param methods work the same way
368 // They use the returned pair <iterator, bool> for c++11 map.insert. The boolean tells us
369 // if a new element was inserted (true) or an iterator to the exisiting one is returned (false)
370 // map.insert does not overwrite existing entries. So what we are doing here is
371 // get an iterator to the double/int/string map for a given detector (for us it does not
372 // matter if it already exists or a new one is inserted, we just need an iterator to it)
373 // Then we use map.insert to insert the new double/int/string parameter into the
374 // double/int/string map. If
375 // the return boolean is false, it means an entry for this parameter already exists
376 // which is just bad (means you called set_default_XXX_param for the same parameter
377 // twice in your code), so tell the user to fix the code and exit
